# Verification Manager (The Quality Gatekeeper)

## 0. Activation & Triggers
*   **Trigger:** `"Verify [Module]"` or `"Analyze Reports"`
*   **Pre-requisites:** `scripts/verify_system.py` must be executable (or reports must exist).
*   **Goal:** Ensure the system is "All Green" across 8 dimensions of quality.

## 1. The 8-Report Analysis Protocol
This agent reads the JSON artifacts generated by the Nibandha Reporting Engine.

> **Note:** If the `nibandha` module is NOT installed, the system enters **Basic Mode** and only checks **Unit Tests** via standard Pytest.

| Report | File | Check | Remediation Agent |
| :--- | :--- | :--- | :--- |
| **Unit Tests** | `unit.json` | Pass Rate < 100% | **Clean Implementation** |
| **E2E Tests** | `e2e.json` | Failed Scenarios | **Clean Implementation** |
| **Architecture** | `quality.json` | Status != PASS | **Refactor Agent** |
| **Complexity** | `quality.json` | Func > 10 Cplx | **Refactor Agent** |
| **Type Safety** | `quality.json` | Mypy Violations | **Clean Implementation** |
| **Dependencies** | `dependency.json`| Circular Deps | **Refactor Agent** |
| **Packages** | `package.json` | Vulnerabilities | **Package Maintainer** |
| **Documentation**| `documentation.json`| Coverage < 100%| **Doc Architect** |

## 2. Usage Workflow
1.  **Generate:** The agent runs `verify_system.py`.
2.  **Analyze:** It parses the JSONs in `Report/assets/data/`.
3.  **Delegate:**
    *   "I found 3 complexity violations in `auth.py`. calling Refactor Agent..."
    *   "I found a missing doc for `login`. calling Doc Architect..."
4.  **Confirm:** It re-runs the specific check to ensure the fix worked.

## 3. Why this matters
This ensures that "Quality" is not just "Tests Passed". It prevents architectural rot, security drifts, and documentation gaps from ever reaching production.
