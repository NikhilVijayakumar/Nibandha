# Export Module: Technical Design

## Design Decisions
The Export module utilizes a **Strategy Pattern** to handle different file formats. This allows new formats to be added without modifying the core logic.

- **Pandoc Integration**: We leverage `pypandoc` (or system pandoc) for robust conversion quality, especially for DOCX/PDF.
- **HTML Fallback**: If pandoc is unavailable, a simpler HTML exporter (using `markdown` library) serves as a fallback.

## Data Flow
1. **Input**: Markdown files generated by `Reporting` module.
2. **Process**:
    - `Exporter` identifies the target format.
    - Selects appropriate strategy (e.g., `PandocStrategy`, `HtmlStrategy`).
    - Applies `default.css` for styling.
3. **Output**: Converted file in `reports/exports/{format}/`.
