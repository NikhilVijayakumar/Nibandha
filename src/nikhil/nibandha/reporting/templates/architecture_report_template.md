## Nomenclature

| Term | Definition |
|------|------------|
| **Clean Architecture** | A software design philosophy that separates concerns into independent layers |
| **Layer** | A logical grouping of code with specific responsibilities (e.g., domain, infrastructure, application) |
| **Dependency Rule** | The principle that dependencies should only point inward (infrastructure ‚Üí application ‚Üí domain) |
| **Contract** | An architectural constraint defined to enforce layer boundaries |
| **Violation** | An import that breaks the defined architectural contracts |
| **Import Linter** | A tool that validates import dependencies against defined architectural rules |

```ini

name = Domain should not import Infrastructure
type = forbidden
source_modules = 
    your_package.*.domain
forbidden_modules =
    your_package.*.infrastructure
```

```ini
[importlinter]
root_package = your_package
```
